# VoiceInput

Нативное macOS-приложение для голосового ввода по удержанию клавиши.

Текущая конфигурация специально смещена в сторону русского языка и бережного расхода ресурсов:

- язык по умолчанию: `ru-RU`;
- режим `prefer on-device` включён сразу;
- без тяжёлых внешних моделей и без постоянного сетевого запроса на каждую фразу.

Что уже реализовано:

- меню-бар приложение без док-иконки;
- удержание клавиши для режима push-to-talk;
- запись микрофона и распознавание речи;
- автоматическая вставка текста в активное приложение;
- fallback в буфер обмена, если не выдан доступ `Accessibility`;
- настройка клавиши, языка распознавания и режима `prefer on-device`.

## Почему не Grok

Я оставил проект полностью бесплатным из коробки. По актуальной документации xAI у Grok есть voice API, но он платный, поэтому базовая реализация здесь сделана на системном `Speech` framework macOS. Для задачи "русский + мало ресурсов" это сейчас наиболее практичный стартовый вариант.

Если позже захочется, сюда несложно добавить отдельный backend для xAI API.

## Сборка

В папке проекта:

```bash
./build.sh
```

Готовое приложение появится тут:

```bash
build/VoiceInput.app
```

## Первый запуск

После запуска приложению понадобятся разрешения:

- `Microphone`
- `Speech Recognition`
- `Accessibility`

Без `Accessibility` распознавание всё равно работает, но текст будет складываться в буфер обмена вместо автоматической печати.

## Управление

- По умолчанию триггер: `Правый Option`
- Альтернативы в меню: `Правый Command`, `F18`, `F19`
- Язык по умолчанию: `ru-RU`

## Ограничения текущей среды

В этой песочнице `swiftc` может упираться в недоступный системный cache path. В `build.sh` это уже обойдёно через локальный cache внутри `build/`.

Ещё один практический момент: если на Mac сломан `Command Line Tools` и даже `clang --version` падает, сборка не дойдёт до конца. В таком случае нужно переустановить CLT или поставить полный Xcode.
